# Руководство по интеграции Модуля 1 (NLP-Парсер чатов)

Привет! Я Antigravity, который делает Frontend и инфраструктуру модулей 2, 3 и 4 для дашборда директора (Кейс №2). 
Твоя задача — реализовать **Модуль 1: NLP-Парсер рабочих чатов (Убийца WhatsApp-рутины)**. 

Для того чтобы мы на защите хакатона показали бесшовную интеграцию между нашими модулями, пожалуйста, следуй этому контракту API и структуре данных. Мои компоненты уже ожидают данные именно в таком формате.

## 🎯 Что тебе нужно сделать

Твой модуль должен принимать неструктурированные текстовые или голосовые сообщения из Telegram-бота (имитация WhatsApp).
Пример входящего сообщения: *"1А - 25 детей, 2 болеют, 1 на соревнованиях. Ахмет сломал парту в 12 кабинете"*.

Твоя LLM/RAG логика должна распарсить этот текст и разбить его на 2 независимых потока (Инциденты и Статистика по питанию).

## 🔌 API Контракт для интеграции с Frontend

Мой компонент **`Reports.tsx`** и **`TaskDistribution.tsx`** (комнаты дашборда) будут отправлять `GET`/`POST` запросы на твой бэкенд, чтобы забрать распарсенные данные. Тебе нужно создать два эндпоинта (или экспортировать функции, если мы в одном монорепозитории).

### 1. Эндпоинт для Отчета по Питанию
`GET /api/v1/nutrition/today`

Мой UI ИИ-Завуча уже нарисован. В нем сейчас стоит мок-задержка, и он "якобы" запрашивает параметры. Когда ты доделаешь АПИ, он будет стучаться к тебе и ожидать такой JSON:

```json
{
  "date": "2026-04-18",
  "totalVseobuch": 250, 
  "absentDetails": {
    "sick_count": 10,
    "competition_count": 5
  },
  "rawMessagesParsed": 14,
  "status": "success"
}
```
*Где `totalVseobuch` ты можешь брать из константы или БД, а `sick_count` и `competition_count` — это те самые данные, которые твоя модель вытащила из чатов за сегодня.*

### 2. Эндпоинт для Инцидентов (Завхоз)
`GET /api/v1/incidents/active` (или Webhook)

Ты распознаешь жалобы из чата и формируешь массив инцидентов. Выдавай такой JSON:

```json
{
  "incidents": [
    {
      "id": "inc-001",
      "timestamp": "2026-04-18T08:30:00Z",
      "location": "Кабинет 12",
      "description": "Сломана парта",
      "reporter": "Учитель 1А класса",
      "assignedTo": "Завхоз",
      "status": "open"
    }
  ]
}
```

## ⚡️ План твоей работы для хакатона:

1. **Создай Telegram Бота** (BotFather), куда учителя могут скидывать текстом или голосовухами свои посещаемости.
2. **Интегрируй Groq / Gemini** (лучше Groq для скорости). Заставь его парсить приходящее сообщение в строгий JSON-объект. Промпт должен быть жестким (System Role: "You are an entity extractor for a school... Return strict JSON...").
3. **Храни данные**. Складывай результаты парсинга за сегодня в `sqlite` / `PostgreSQL` / `temp.json`.
4. **Отдай по REST**. Разверни легкий FastAPI или Express сервер, который будет отдавать JSON по форматам выше, чтобы мой Frontend смог сделать `fetch('http://localhost:8000/api/v1/nutrition/today')` при открытии вкладки.

Удачи! Если что-то нужно поменять в контракте данных — скажи пользователю, я быстро перепишу типы на Фронтенде.
